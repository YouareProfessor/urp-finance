# URP 재무 보드

URP(니가 교수) 팀의 재무관리 프로그램. 수익 시뮬레이터 · 고정지출 · 월별 손익표 · 엑셀 연동을 한 화면에서, 팀 전원이 실시간으로 함께 봅니다.

- 배포: GitHub Pages (정적 사이트, 빌드 없음)
- 데이터: Firebase Firestore (yourprofessor-94a2d) 실시간 동기화
- 디자인: URP 소개서 웹과 동일한 톤 (종이 질감 + 브랜드 오렌지)

## 접근 방법

팀 공용 비밀번호 하나로 잠급니다. 비밀번호를 SHA-256으로 해시한 값이 데이터 방(경로)이 되므로, 비밀번호 없이는 데이터 위치 자체를 알 수 없습니다.

- 비밀번호는 **긴 문장형**을 추천합니다 (예: 단어 3개 이상 조합).
- 비밀번호를 바꾸면 새 방이 생기므로, 바꾸기 전에 엑셀로 내보내고 새 방에서 가져오면 됩니다.
- 이 저장소는 공개여도 안전하지만, **비밀번호는 절대 커밋·공개 채널에 적지 마세요.**

## 처음 방 만들기 (1회, 운영자만)

1. `index.html?setup=1` 로 접속
2. 팀과 공유할 비밀번호 입력 → "새 보드 만들기" 확인
3. 이후 팀원들은 일반 주소에서 같은 비밀번호로 입장

## 인건비 원칙

개인별 급여는 절대 항목화하지 않습니다. "인건비 합계" 한 줄만 존재하며(삭제 불가), 엑셀 가져오기에서 인건비로 분류된 행도 자동으로 합계 한 줄로 합산됩니다.

## Firestore 규칙

`docs/firestore-rules-snippet.txt` 블록이 회사 규칙 파일(workspace 저장소의 `firestore.rules`)에 추가되어 있어야 합니다. 규칙 배포는 workspace 폴더에서:

```
firebase deploy --only firestore:rules --project yourprofessor-94a2d
```

⚠️ 규칙은 프로젝트당 1세트 — 반드시 통합본 전체를 배포할 것 (부분 배포 시 다른 앱이 깨진 전례 있음).
⚠️ 이 프로젝트에 익명 인증(Anonymous Auth)을 켜지 마세요 — 기존 워크스페이스 데이터가 익명 사용자에게 열립니다.

## 파일 구조

```
index.html      앱 셸 (잠금 화면 + 탭 5개)
css/tokens.css  디자인 토큰 (소개서 팔레트 + CVD 검증 차트 팔레트)
css/app.css     레이아웃·컴포넌트
js/calc.js      순수 계산 (node로 테스트 가능)
js/auth.js      비밀번호 → 방 파생, 잠금
js/store.js     실시간 상태 + 저장
js/charts.js    SVG 차트 (콤보/멀티라인/도넛)
js/xlsx.js      SheetJS 내보내기/가져오기
js/ui-*.js      화면 5개
js/main.js      부팅·라우터
```
